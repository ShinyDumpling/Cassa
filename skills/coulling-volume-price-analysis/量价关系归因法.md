# 量价关系归因法

## 一、方法目标

量价关系归因法用于解释最近几根 K 线中价格变化与成交量变化之间的关系。

它不依赖固定形态名称，而是按照下面的顺序推理：

```text
价格变化
-> 成交量变化
-> 判断买方力量和卖方压力的相对变化
-> 输出看多或看空及结论强度
```

这里的“看多 / 看空”是量价关系方向判断，不等同于直接买入或卖出建议。

## 二、理论基础

本方法主要结合：

- 威科夫投入—产出定律：成交量是投入，价格变化是产出；量价匹配是确认，量价不匹配是异常。
- 供求关系：价格结果需要结合需求和供给的相对变化解释。
- VPA / VSA：结合价格方向、实体、影线、收盘位置和成交量观察供给与需求。

## 三、价格特征识别

不能只看单根 K 线。原则上比较最近连续 2～3 根或更多 K 线，观察以下维度：

- 方向：上涨、下跌、横盘；
- 速度：加速、放缓、平稳；
- 波动：扩大、收窄、剧烈震荡；
- 实体：实体扩大、缩小、稳定；
- 影线：上影线变长、下影线变长、影线收窄；
- 收盘位置：接近最高、接近最低、位于中部；
- 连续性：连续上涨、连续下跌、涨跌交替、趋势中断。

价格特征不限制为固定四种，应从上述维度灵活组合成自然语言，例如：

```text
上涨趋势中的涨幅放缓，且阳线实体持续缩小
下跌趋势中的跌幅收窄，同时下影线变长
价格横盘，K 线实体缩小，成交量持续下降
价格上涨加速，但实体没有同步扩大
```

如果输入数据不足以识别价格特征，仍然必须输出看多或看空，但要降低结论强度，并在证据中说明样本不足。

## 四、双因归因

常见价格特征与候选原因：

| 价格特征 | 候选原因 A | 候选原因 B |
|---|---|---|
| 涨幅放缓 | 买方减弱 | 卖方增加 |
| 涨幅加速 | 买方增加 | 卖方减弱 |
| 跌幅放缓 | 买方增加 | 卖方减弱 |
| 跌幅加速 | 买方减弱 | 卖方增加 |

对于横盘、影线变化、实体变化等复合特征，提出最接近的买方 / 卖方候选原因，不强行套用固定形态名称。

## 五、成交量验证规则

本方法只使用：

```text
daily_kline[].volume
```

禁止使用 `daily_kline[].volume_ratio` 对本模块的候选原因做判断。

成交量可以观察：

- 放量；
- 缩量；
- 平量；
- 连续放量；
- 连续缩量；
- 先放量后缩量；
- 先缩量后放量。

每一笔成交天然同时包含买方和卖方，因此不能把成交量直接等同于单独的买方或卖方变化：

```text
成交量放大 != 单独确认买方增加
成交量萎缩 != 单独确认买方减弱
```

成交量只能说明交易意愿、市场参与度或筹码交换强度发生变化。最终判断哪一方更占优势，必须结合价格方向、实体、影线、收盘位置和连续 K 线共同判断。

典型解释：

```text
上涨 + 放量 + 大实体
-> 更支持买方力量增强并占据主动

上涨 + 放量 + 小实体或长上影
-> 更支持卖方压力增强，上涨推进被抵消

下跌 + 缩量 + 跌幅收窄
-> 更支持卖方压力减弱
```

## 六、买卖方归因边界规则

不能把成交量直接等同于单独的买方或卖方变化，更不能用于推断买卖人数：

```text
成交量放大 != 单独确认买方增加
成交量萎缩 != 单独确认买方减弱
成交量不能用于判断买家或卖家的实际人数
```

量价结果只能表达为"更支持买方力量增强/减弱"或"更支持卖方压力增强/减弱"。当价格、实体、影线和成交量不能区分哪一方发生变化时，必须写明"买卖双方交换增强/减弱，暂不能唯一归因"，然后根据连续 K 线给出较弱的看多或看空倾向。

## 七、看多 / 看空规则

模块必须输出看多或看空，不能只输出"无法判断"。证据不足时，输出倾向方向并降低结论强度。

基础规则：

| 价格表现 | 成交量表现 | 方向倾向 |
|---|---|---|
| 上涨加速 | 放量 | 看多 |
| 上涨加速 | 缩量 | 看多，但警惕虚涨 |
| 涨幅放缓 | 缩量 | 看空 |
| 涨幅放缓 | 放量 | 看空，警惕派发 |
| 下跌加速 | 放量 | 看空 |
| 下跌加速 | 缩量 | 看空，但下跌动能可能不足 |
| 跌幅放缓 | 缩量 | 看多 |
| 跌幅放缓 | 放量 | 根据实体、影线和后续价格结果选择方向，降低结论强度 |
| 横盘 | 缩量 | 偏多，等待测试 |
| 横盘 | 放量 | 偏空，警惕筹码交换或派发 |

## 八、盘中规则

```text
价格变化：最近三根 K 线仍在上涨，但涨幅连续收窄，最新收盘价为 10.86 元（2026-07-17 日收盘价）。
成交量变化：最近三根 K 线成交量连续缩小。
买卖方结果：成交活跃度下降且价格推进放缓，更支持买方力量减弱；现有数据不能证明买家人数减少。
量价结论：看空，结论强度为中。
```

## 八、盘中规则

当 `market_context.is_intraday == true` 时，最新一根 K 线的 `volume` 是盘中累计成交量，不能与历史完整日 K 的全天成交量直接等价比较。

此时：

- 仍然可以输出看多或看空；
- 结论必须说明是盘中暂态判断；
- 结论强度应适当降低；
- 不得写成已经确认突破、已经确认反转或已经完成放量止涨 / 止跌。

推荐表达：

```text
盘中暂时表现为量价偏多；由于最新成交量仍为累计值，收盘后仍需确认。
```

## 九、输出要求

该方法对应的输出模块必须使用普通模块结构：

```json
"volume_price_relation": {
  "result": "",
  "data": [
    {
      "kline": {},
      "evidence": ""
    }
  ],
  "refs": []
}
```

`result` 必须严格按照以下顺序输出：

1. `价格变化：`说明连续 K 线的方向、速度、实体、影线和收盘位置变化。
2. `成交量变化：`说明放量、缩量、平量或连续变化。
3. `买卖方结果：`说明更支持买方力量或卖方压力增强/减弱；不得推断买家或卖家人数。
4. `量价结论：`明确输出看多或看空，并说明结论强度。

`result` 和 `data[].evidence` 中出现价格时，必须标注交易日与价格字段、区间统计来源、筹码字段或派生规则。

`result` 和 `data[].evidence` 不得使用"供给、需求"描述买卖方变化；理论索引中的原书理论名称不受此限制。

`data[].kline` 必须从输入 `daily_kline` 中原样复制，不能手写、重算、四舍五入或拼接字段。
