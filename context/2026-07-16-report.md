# Report：结构化 JSON 输出 Markdown 报告方案

## 目标

新增一个通用的 `report.py`，将业务模块产生的结构化 JSON 渲染成 Markdown 文件，并统一写入 Cassa 项目的 `result/` 目录。

第一版先服务 `thesis / thises` 量价分析报告，后续选股等模块也可以复用同一个报告输出脚本。

## 第一版范围

第一版只实现以下功能：

1. 接收已经解析好的 JSON 数据对象。
2. 接收一个报告类型参数。
3. 根据报告类型选择 Markdown 模板。
4. 使用 Jinja2 渲染 Markdown。
5. 将结果直接写入 `result/` 目录。
6. 返回最终生成文件的路径。

第一版暂不实现：

- CLI 入口。
- JSON 文件路径输入。
- Adapter 或中间 `report_context` 数据层。
- 主模板与模块模板拆分。
- 自定义 Jinja2 filter。
- `tabulate` 表格工具。
- Markdown 转 HTML、PDF 或其他格式。
- 自动摘要或额外业务推理。

## 设计原则

### 1. report.py 只负责报告输出

业务模块负责产生 JSON，`report.py` 不负责：

- 股票分析。
- 重新计算指标。
- 修改分析结论。
- 根据多个字段重新推导新的业务判断。

`report.py` 只负责把 JSON 中已有的内容组织成 Markdown 文档。

### 2. 直接从 JSON 读取字段

第一版不增加中间数据适配层。

模板直接读取 JSON 字段，例如：

```jinja2
{{ data.name }}
{{ data.code }}
{{ data.direction.result }}
```

这样可以保持实现简单，也能清楚看到模板和 JSON 字段之间的对应关系。

如果未来原始 JSON 和展示结构差异很大，再考虑增加中间上下文层。

### 3. 报告类型暂时使用判断

当前模板数量较少，先在 `report.py` 中使用简单判断选择模板，不提前设计注册器或插件机制。

目前支持：

```text
thesis -> templates/thesis.md.j2
thises -> templates/thesis.md.j2
```

`thises` 作为兼容别名，避免业务模块名称和报告名称不一致时无法调用。

以后新增选股报告时，再增加：

```text
stock_selection -> templates/stock_selection.md.j2
```

### 4. 一个报告类型对应一个模板

第一版不拆分主模板和模块模板。

目录结构：

```text
Cassa/
  report.py
  templates/
    thesis.md.j2
  result/
```

`thesis.md.j2` 中直接包含完整报告的所有章节。

### 5. 不新增模板工具

第一版不注册任何自定义 filter 或函数。

表格直接使用 Jinja2 循环生成 Markdown 表格；数字暂时保持 JSON 原始值，不在报告层改变小数位、单位或百分比口径。

```python
"""将结构化 JSON 渲染为 Markdown 报告并写入 result 目录。"""

from __future__ import annotations

import re
from datetime import datetime
from pathlib import Path
from typing import Any

from jinja2 import Environment, FileSystemLoader, StrictUndefined


# report.py 所在目录，也就是 Cassa 项目根目录。
PROJECT_ROOT = Path(__file__).resolve().parent
TEMPLATES_DIR = PROJECT_ROOT / "templates"
RESULT_DIR = PROJECT_ROOT / "result"

# 第一版模板数量较少，直接使用映射表选择模板。
TEMPLATE_MAP = {
    "thesis": "thesis.md.j2",
    "thises": "thesis.md.j2",
}


def build_template_environment() -> Environment:
    """创建 Jinja2 模板环境，返回用于渲染 Markdown 的环境对象。"""
    return Environment(
        loader=FileSystemLoader(str(TEMPLATES_DIR)),
        undefined=StrictUndefined,
        trim_blocks=True,
        lstrip_blocks=True,
        keep_trailing_newline=True,
    )


def sanitize_filename_part(value: Any) -> str:
    """清理文件名片段中的非法字符，返回适合 Windows 文件名的文本。"""
    text = str(value or "").strip()
    text = re.sub(r'[<>:"/\\|?*\x00-\x1f]', "_", text)
    return text or "unknown"


def build_output_path(
    data: dict[str, Any],
    report_type: str,
    output_name: str | None = None,
) -> Path:
    """根据报告数据生成 result 目录下的输出路径，不覆盖已有报告。"""
    RESULT_DIR.mkdir(parents=True, exist_ok=True)

    if output_name:
        filename = sanitize_filename_part(output_name)
        if not filename.lower().endswith(".md"):
            filename = f"{filename}.md"
    else:
        code = sanitize_filename_part(data.get("code"))
        name = sanitize_filename_part(data.get("name"))
        timestamp = datetime.now().strftime("%Y%m%d-%H%M%S")
        filename = f"{report_type}-{code}-{name}-{timestamp}.md"

    output_path = RESULT_DIR / filename
    if output_path.exists() and not output_name:
        timestamp = datetime.now().strftime("%Y%m%d-%H%M%S-%f")
        output_path = RESULT_DIR / (
            f"{report_type}-{sanitize_filename_part(data.get('code'))}-"
            f"{sanitize_filename_part(data.get('name'))}-{timestamp}.md"
        )

    return output_path


def generate_report(
    data: dict[str, Any],
    report_type: str,
    output_name: str | None = None,
) -> Path:
    """根据 JSON 数据和报告类型渲染 Markdown，并写入 result 目录。

    输入：
        data：已经解析好的 JSON 对象。
        report_type：报告类型，例如 thesis 或 thises。
        output_name：可选输出文件名，不传时自动生成文件名。

    输出：
        实际生成的 Markdown 文件路径。
    """
    if not isinstance(data, dict):
        raise TypeError("报告输入必须是解析后的 JSON 对象，即 Python dict。")

    normalized_report_type = str(report_type or "").strip().lower()
    template_name = TEMPLATE_MAP.get(normalized_report_type)
    if not template_name:
        supported_types = ", ".join(sorted(TEMPLATE_MAP))
        raise ValueError(
            f"不支持的报告类型：{report_type}。当前支持：{supported_types}"
        )

    template_path = TEMPLATES_DIR / template_name
    if not template_path.is_file():
        raise FileNotFoundError(f"报告模板不存在：{template_path}")

    template_environment = build_template_environment()
    template = template_environment.get_template(template_name)
    markdown_text = template.render(data=data)

    output_path = build_output_path(data, normalized_report_type, output_name)
    output_path.write_text(markdown_text, encoding="utf-8")
    return output_path


__all__ = ["generate_report"]
```

## 调用方式

业务代码已经拿到 JSON 对象后，直接调用：

```python
from report import generate_report

output_path = generate_report(
    data=thises_analysis_json,
    report_type="thesis",
)

print(output_path)
```

默认输出到：

```text
Cassa/result/thesis-代码-名称-时间.md
```

也可以指定文件名：

```python
output_path = generate_report(
    data=thises_analysis_json,
    report_type="thesis",
    output_name="002546.SZ-新联电子.md",
)
```

8. 输入 JSON 中的分析结论和证据文本没有被报告层重新改写。
