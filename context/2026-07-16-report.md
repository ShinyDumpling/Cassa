# Report：结构化 JSON 输出 Markdown 报告方案

## 目标

新增一个通用的 `report.py`，将业务模块产生的结构化 JSON 原样保存为 JSON 文件，同时渲染为 Markdown 报告，并统一写入 Cassa 项目的 `result/` 目录。

第一版先服务 `thesis / thises` 量价分析报告，后续选股等模块也可以复用同一个报告输出脚本。

## 第一版范围

第一版只实现以下功能：

1. 接收已经解析好的 JSON 数据对象。
2. 接收一个报告类型参数。
3. 根据报告类型选择 Markdown 模板。
4. 使用 Jinja2 渲染 Markdown。
5. 将 JSON 和 Markdown 直接写入 `result/` 目录。
6. 返回两个最终生成文件的路径。

第一版暂不实现：

- CLI 入口。
- JSON 文件路径输入。
- Adapter 或中间 `report_context` 数据层。
- 主模板与模块模板拆分。
- 自定义 Jinja2 filter。
- `tabulate` 表格工具。
- Markdown 转 HTML、PDF 或其他格式。
- 自动摘要或额外业务推理。

## 设计原则

### 1. report.py 只负责报告输出

业务模块负责产生 JSON，`report.py` 不负责：

- 股票分析。
- 重新计算指标。
- 修改分析结论。
- 根据多个字段重新推导新的业务判断。

`report.py` 只负责保存输入 JSON，并把 JSON 中已有的内容组织成 Markdown 文档。

### 2. 直接从 JSON 读取字段

第一版不增加中间数据适配层。

模板直接读取 JSON 字段，例如：

```jinja2
{{ data.name }}
{{ data.code }}
{{ data.direction.result }}
```

这样可以保持实现简单，也能清楚看到模板和 JSON 字段之间的对应关系。

如果未来原始 JSON 和展示结构差异很大，再考虑增加中间上下文层。

### 3. 报告类型暂时使用判断

当前模板数量较少，先在 `report.py` 中使用简单判断选择模板，不提前设计注册器或插件机制。

目前支持：

```text
thesis -> templates/thesis.md.j2
thises -> templates/thesis.md.j2
```

`thises` 作为兼容别名，避免业务模块名称和报告名称不一致时无法调用。

以后新增选股报告时，再增加：

```text
stock_selection -> templates/stock_selection.md.j2
```

### 4. 一个报告类型对应一个模板

第一版不拆分主模板和模块模板。

目录结构：

```text
Cassa/
  report.py
  templates/
    thesis.md.j2
  result/
```

`thesis.md.j2` 中直接包含完整报告的所有章节。

### 5. 不新增模板工具

第一版不注册任何自定义 filter 或函数。

表格直接使用 Jinja2 循环生成 Markdown 表格；数字暂时保持 JSON 原始值，不在报告层改变小数位、单位或百分比口径。

```python
"""将结构化 JSON 保存为数据文件，并渲染为 Markdown 报告。"""

from __future__ import annotations

import re
import json
from datetime import datetime
from pathlib import Path
from typing import Any

from jinja2 import Environment, FileSystemLoader, StrictUndefined


# report.py 所在目录，也就是 Cassa 项目根目录。
PROJECT_ROOT = Path(__file__).resolve().parent
TEMPLATES_DIR = PROJECT_ROOT / "templates"
RESULT_DIR = PROJECT_ROOT / "result"

# 第一版模板数量较少，直接使用映射表选择模板。
TEMPLATE_MAP = {
    "thesis": "thesis.md.j2",
    "thises": "thesis.md.j2",
}


def build_template_environment() -> Environment:
    """创建 Jinja2 模板环境，返回用于渲染 Markdown 的环境对象。"""
    return Environment(
        loader=FileSystemLoader(str(TEMPLATES_DIR)),
        undefined=StrictUndefined,
        trim_blocks=True,
        lstrip_blocks=True,
        keep_trailing_newline=True,
    )


def sanitize_filename_part(value: Any) -> str:
    """清理文件名片段中的非法字符，返回适合 Windows 文件名的文本。"""
    text = str(value or "").strip()
    text = re.sub(r'[<>:"/\\|?*\x00-\x1f]', "_", text)
    return text or "unknown"


def build_output_paths(
    data: dict[str, Any],
    report_type: str,
    timestamp: datetime,
) -> tuple[Path, Path]:
    """生成一组不覆盖旧文件的 JSON 和 Markdown 输出路径。"""
    RESULT_DIR.mkdir(parents=True, exist_ok=True)

    code = sanitize_filename_part(data.get("code"))
    name = sanitize_filename_part(data.get("name"))
    base_name = (
        f"{report_type}-{code}-{name}-"
        f"{timestamp.strftime('%Y%m%d-%H%M%S')}"
    )

    candidate_suffixes = ["", f"-{timestamp.strftime('%f')}"]
    candidate_suffixes.extend(f"-{index}" for index in range(2, 1000))

    for suffix in candidate_suffixes:
        data_path = RESULT_DIR / f"{base_name}{suffix}-data.json"
        report_path = RESULT_DIR / f"{base_name}{suffix}-report.md"
        if not data_path.exists() and not report_path.exists():
            return data_path, report_path

    raise FileExistsError("无法生成不重复的报告文件名。")


def generate_report_bundle(
    data: dict[str, Any],
    report_type: str,
) -> dict[str, Path]:
    """保存原始 JSON 和 Markdown 报告，返回两个文件路径。

    输入：
        data：已经解析好的 JSON 对象。
        report_type：报告类型，例如 thesis 或 thises。

    输出：
        包含 data_path 和 report_path 的路径字典。
    """
    if not isinstance(data, dict):
        raise TypeError("报告输入必须是解析后的 JSON 对象，即 Python dict。")

    normalized_report_type = str(report_type or "").strip().lower()
    template_name = TEMPLATE_MAP.get(normalized_report_type)
    if not template_name:
        supported_types = ", ".join(sorted(TEMPLATE_MAP))
        raise ValueError(
            f"不支持的报告类型：{report_type}。当前支持：{supported_types}"
        )

    template_path = TEMPLATES_DIR / template_name
    if not template_path.is_file():
        raise FileNotFoundError(f"报告模板不存在：{template_path}")

    template_environment = build_template_environment()
    template = template_environment.get_template(template_name)
    markdown_text = template.render(data=data)

    data_path, report_path = build_output_paths(
        data,
        normalized_report_type,
        datetime.now(),
    )

    try:
        with data_path.open("x", encoding="utf-8", newline="\n") as file:
            json.dump(data, file, ensure_ascii=False, indent=2)
            file.write("\n")

        with report_path.open("x", encoding="utf-8", newline="\n") as file:
            file.write(markdown_text)
    except Exception:
        if data_path.exists() and not report_path.exists():
            data_path.unlink()
        raise

    return {
        "data_path": data_path,
        "report_path": report_path,
    }


__all__ = ["generate_report_bundle"]
```

## 2026-07-20 更新：按报告类型和生成日期整理 thesis 输出目录

### 一、问题与原因

当前 `report.py` 将所有报告文件直接写入：

```text
Cassa/result/
```

随着不同股票、不同日期和不同报告类型持续生成，`result/` 会混杂大量 `data.json` 和 `report.md` 文件，带来以下问题：

1. 无法按报告类型快速定位文件。
2. 无法按生成日期浏览某一天产生的报告。
3. 同一股票的多次分析结果与其他股票文件混在一起。
4. `thises-volume-price` 的报告输出协议仍然描述为直接写入 `result/`，与新的归档意图不一致。

本次只调整输出目录组织方式，不改变以下既有行为：

- JSON 和 Markdown 仍然由同一次 `generate_report_bundle()` 生成。
- 两个文件继续使用完全相同的主体名称和时间戳。
- 每次生成仍然禁止覆盖已有文件。
- JSON 内容、Markdown 模板和报告业务结论不变。
- 已有 `result/` 根目录中的旧文件不自动搬迁、不删除。

### 二、目标目录结构

新的 thesis 报告目录结构为：

```text
Cassa/
└── result/
    └── thesis/
        ├── 2026-07-20/
        │   ├── thesis-002546.SZ-新联电子-20260720-145045-data.json
        │   └── thesis-002546.SZ-新联电子-20260720-145045-report.md
        └── 2026-07-21/
            ├── thesis-002546.SZ-新联电子-20260721-091530-data.json
            └── thesis-002546.SZ-新联电子-20260721-091530-report.md
```

目录规则：

```text
result/{normalized_report_type}/{生成日期}/
```

对于当前 thesis 流程：

```text
result/thesis/YYYY-MM-DD/
```

其中：

- `normalized_report_type` 使用报告类型的规范名称；`thises` 兼容别名仍归一化为 `thesis`。
- `生成日期` 来自本次报告生成时的本地时间，格式为 `YYYY-MM-DD`。
- 文件名中的时间戳与日期目录使用同一个 `datetime` 对象生成，避免跨日或时间不一致。
- 同一天同一秒内重复生成时，沿用现有的微秒或递增序号防覆盖机制。

### 三、技术方案

#### 1. 由 `report.py` 统一决定目录

调用方继续使用原有接口：

```python
from report import generate_report_bundle

paths = generate_report_bundle(data, "thesis")
```

调用方不传入输出目录，也不自行拼接文件名。这样可以保证所有报告类型都遵守统一的目录、命名和防覆盖规则。

#### 2. 报告类型使用白名单映射

不能直接把用户传入的 `report_type` 拼接成路径。应先通过 `TEMPLATE_MAP` 验证类型，再通过明确的目录映射生成路径，避免非法报告类型或路径字符影响输出位置。

当前映射为：

```python
REPORT_DIR_MAP = {
    "thesis": RESULT_DIR / "thesis",
    "thises": RESULT_DIR / "thesis",
}
```

归一化后，`thises` 和 `thesis` 都写入 `result/thesis/`。

#### 3. 保持成对文件的原子性约束

执行顺序保持为：

```text
1. 校验输入数据和报告类型。
2. 在内存中完成 Markdown 渲染。
3. 生成 result/thesis/YYYY-MM-DD/ 目录。
4. 生成一组不冲突的共同文件名。
5. 独占创建 JSON 文件。
6. 独占创建 Markdown 文件。
7. Markdown 写入失败时删除本次刚创建的 JSON 文件。
8. 返回两个实际文件路径。
```

### 四、`report.py` 完整替换代码

将 `D:\股神养成plan\Cassa\report.py` 替换为以下完整代码：

```python
"""将结构化 JSON 保存为数据文件，并渲染为 Markdown 报告。"""

from __future__ import annotations

import json
import re
from datetime import datetime
from pathlib import Path
from typing import Any

from jinja2 import Environment, FileSystemLoader, StrictUndefined


PROJECT_ROOT = Path(__file__).resolve().parent
TEMPLATES_DIR = PROJECT_ROOT / "templates"
RESULT_DIR = PROJECT_ROOT / "result"

TEMPLATE_MAP = {
    "thesis": "thesis.md.j2",
    "thises": "thesis.md.j2",
}

# 报告类型到归档目录的白名单映射。
# thises 是兼容别名，实际统一归档到 result/thesis/。
REPORT_DIR_MAP = {
    "thesis": RESULT_DIR / "thesis",
    "thises": RESULT_DIR / "thesis",
}


def build_template_environment() -> Environment:
    """创建 Jinja2 模板环境，返回用于渲染 Markdown 的环境对象。"""
    return Environment(
        loader=FileSystemLoader(str(TEMPLATES_DIR)),
        undefined=StrictUndefined,
        trim_blocks=True,
        lstrip_blocks=True,
        keep_trailing_newline=True,
    )


def sanitize_filename_part(value: Any) -> str:
    """清理文件名片段中的非法字符，返回适合 Windows 文件名的文本。"""
    text = str(value or "").strip()
    text = re.sub(r'[<>:"/\\|?*\x00-\x1f]', "_", text)
    return text or "unknown"


def build_output_paths(
    data: dict[str, Any],
    report_type: str,
    timestamp: datetime,
) -> tuple[Path, Path]:
    """生成按报告类型和生成日期归档、且不覆盖旧文件的输出路径。"""
    report_root = REPORT_DIR_MAP[report_type]
    date_dir = report_root / timestamp.strftime("%Y-%m-%d")
    date_dir.mkdir(parents=True, exist_ok=True)

    code = sanitize_filename_part(data.get("code"))
    name = sanitize_filename_part(data.get("name"))
    base_name = (
        f"{report_type}-{code}-{name}-"
        f"{timestamp.strftime('%Y%m%d-%H%M%S')}"
    )

    candidate_suffixes = ["", f"-{timestamp.strftime('%f')}"]
    candidate_suffixes.extend(f"-{index}" for index in range(2, 1000))

    for suffix in candidate_suffixes:
        data_path = date_dir / f"{base_name}{suffix}-data.json"
        report_path = date_dir / f"{base_name}{suffix}-report.md"
        if not data_path.exists() and not report_path.exists():
            return data_path, report_path

    raise FileExistsError("无法生成不重复的报告文件名。")


def generate_report_bundle(
    data: dict[str, Any],
    report_type: str,
) -> dict[str, Path]:
    """保存原始 JSON 和 Markdown 报告，返回两个实际文件路径。"""
    if not isinstance(data, dict):
        raise TypeError("报告输入必须是解析后的 JSON 对象，即 Python dict。")

    normalized_report_type = str(report_type or "").strip().lower()
    template_name = TEMPLATE_MAP.get(normalized_report_type)
    report_root = REPORT_DIR_MAP.get(normalized_report_type)
    if not template_name or report_root is None:
        supported_types = ", ".join(sorted(TEMPLATE_MAP))
        raise ValueError(
            f"不支持的报告类型：{report_type}。当前支持：{supported_types}"
        )

    template_path = TEMPLATES_DIR / template_name
    if not template_path.is_file():
        raise FileNotFoundError(f"报告模板不存在：{template_path}")

    template_environment = build_template_environment()
    template = template_environment.get_template(template_name)
    markdown_text = template.render(data=data)

    timestamp = datetime.now()
    data_path, report_path = build_output_paths(
        data,
        normalized_report_type,
        timestamp,
    )

    try:
        with data_path.open("x", encoding="utf-8", newline="\n") as file:
            json.dump(data, file, ensure_ascii=False, indent=2)
            file.write("\n")

        with report_path.open("x", encoding="utf-8", newline="\n") as file:
            file.write(markdown_text)
    except Exception:
        if data_path.exists() and not report_path.exists():
            data_path.unlink()
        raise

    return {
        "data_path": data_path,
        "report_path": report_path,
    }


__all__ = ["generate_report_bundle"]
```
