# Market Dashboard / 板块轮动 / 证券拉升监控

## 主题概述

这一组讨论主要围绕 `Sentinel` 项目，重点是：

1. `market_dashboard.py` 负责什么
2. 如何观察大盘与板块轮动
3. 如何在板块里理解个股角色
4. 如何做一个足够快、足够简洁的板块异动监控脚本

## market_dashboard 的定位

用户已经在用 `D:\股神养成plan\Sentinel\market_dashboard.py` 来观察：

1. 大盘整体状态
2. 板块强弱
3. 轮动方向

并计划把这些数据提供给 LLM，用于做盘后/盘前预期。

后续还讨论过：

1. `market_dashboard.py` 既要保留肉眼可读输出
2. 又要额外生成一个结构化 JSON 结果，给别的脚本消费
3. 这个 JSON 生成逻辑不依赖 LLM

## 关于字段理解的重要修正

讨论中确认过一个关键字段问题：

1. `fLianB` 实际是量比，不是连板天数
2. `EverZTCount` 才是连板相关字段

这个结论很重要，因为它直接影响大盘/板块/个股描述是否失真。

## 关于板块内个股角色

在“熟悉 market_dashboard.py”的会话里，形成了一个比较清晰的第一版分类想法：

1. 龙头
   - 代表板块最强辨识度和情绪牵引
2. 中军
   - 容量大、稳定性强、对板块方向有定海神针作用
3. 弹性票
   - 波动大、弹性强、适合捕捉超额收益，但也更容易大幅回撤
4. 跟风票
   - 自身辨识度弱，更依赖板块带动

这个分类后来也延续到了“个股预期”讨论里。

## 关于“核心股”的补充想法

用户后来提出过一个很有价值的补充概念：

- 有些股票不是最强涨停龙头，也不是传统意义上的中军，但它几乎可以作为板块温度计。
- 只要它涨，板块通常就涨；它跌，板块通常就弱。

这个概念和“龙头”并不完全相同，更像是“板块风向锚点”。这个想法已被认为有业务意义，适合后续继续打磨。

## 证券板块快速拉升监控

在“监控证券板块快速拉升”的会话中，做过一个简化版监控脚本，目标是：

1. 实时盯证券板块及相关个股
2. 在回调阶段快速发现异动拉升
3. 输出足够简洁，方便盘中一眼看重点

最后确定的控制台表格字段只保留：

1. 名字
2. 代码
3. 涨幅
4. 涨速
5. 现价
6. 相对 MA5

其中：

- `相对 MA5` 用带正负号的百分比显示
- `MA5` 来自本地 `alphasift-fork/data/tdx_daily.db`

这背后的原则是：盘中监控脚本必须把输出尽量压缩到“肉眼可用”的程度，不能一屏全是噪音。

## 另一个小型成果

在 `market_dashboard` 相关会话中，还做过一个独立脚本：

- `D:\股神养成plan\Sentinel\tdx_get_more_info_demo.py`

作用是把通达信 `get_more_info` 的原始结果，整理成更适合人读的一行一字段格式，并加上中文标签，方便后续理解接口能力。

## 当前结论

这一组会话沉淀出来的最关键认识是：

1. 大盘/板块观察和盘中监控必须服务于“快速判断”
2. 输出应该优先考虑肉眼使用体验
3. 板块内个股不能只分“强弱”，还需要有角色感知
4. 结构化 JSON 和肉眼报告应该并存，但职责要分清
