# 选股模块：突破前入场策略

## 主题概述

基于 B 站视频《一个突破三种操作（突破前，突破时，突破后）交易者必学策略》（作者：磐石交易学院，BV1NNS7BmEGh）的策略，在 Cassa 项目中实现选股筛选器。

视频核心观点：书本上只讲了"突破时入场"和"突破回踩入场"两种模式，但遗漏了更重要的"突破之前入场"。突破前入场的前提是先解决"趋势"和"位置"两个底层问题，把趋势和位置纳入突破入场技巧才是完整的入场体系。

## 视频策略拆解

### 三种入场模式及风险

| 模式 | 风险 |
|------|------|
| 突破时入场 | 假突破、买在高位 |
| 突破收线入场 | 假突破、买在高位、止损过大 |
| 突破回踩入场 | 稳健但可能直接起飞不给机会 |
| 突破之前入场 | 风险最低，但需先解决趋势和位置判断 |

### 突破前入场的实现路径

1. 确认趋势反转：打破下跌节奏 + MACD背离 + MACD快慢线站上零轴
2. 判断波段位置：用MACD定位波段（零轴下方金叉=低点，零轴上方死叉=高点，中间盘整=波段中部）
3. 盘整飞吻状态：盘整时MACD在零轴上方贴零轴（飞吻）且没死叉 = 波段中部，方向为多
4. 在突破发生前进场

### 关键概念

- MACD背离：价格创新低但MACD低点抬高，说明下跌动能减弱
- 零轴下方金叉：DIF从下穿上DEA且都在零轴下方，标志波段低点
- 零轴上方死叉：DIF从上下穿DEA且都在零轴上方，标志波段高点
- 飞吻：DIF在零轴上方贴近零轴运行，表示上涨刚启动、动能温和
- 盘整：价格在小幅区间内横盘蓄势
- 单波段 vs 多波段：中部盘整突破属单波段主升浪，高位回调盘整属ABCD多波段，位置不同不能用同样方式入场

## 选股模块设计

### 数据源

- SQLite 数据库：`D:\股神养成plan\Sentinel\all_daily_k.db`
- 表 `daily_kline`：code（纯数字代码）、trade_date、open_price、high_price、low_price、close_price、volume、amount
- 5530只股票，日期范围 2025-09-05 到 2026-07-03，最多197根K线
- MACD 计算使用通达信公式引擎 `formula_process_mul_zb`（批量接口），保证与通达信客户端一致

### 选股流程（4步漏斗）

#### 第1步：底背离检测

- 用 pivot low（左右各5根K线窗口）找价格枢轴低点，取最后一个作为前一个低点
- 最近低点不要求 pivot 确认，直接取最近10根K线的最低价位置（解决 pivot 右侧5根确认导致信号延迟的问题）
- 两个低点间隔不超过40根K线
- 最近低点距最新K线不超过50根（时效性约束）
- 价格必须创新低：最近低点 < 前一个低点
- MACD柱状图值必须抬高：最近低点位置的MACD值 > 前一个低点位置的MACD值

#### 第2步：趋势反转确认

三个子条件同时满足：
- DIF当前在零轴上方（最新DIF > 0）
- 背离低点之后出现过零轴下方金叉（DIF从下穿上DEA，且当时两者都在零轴下方）
- 下跌节奏被打破：最近20根K线的最低价 ≥ 背离低点的价格（不再创新低即可，不要求高于）

#### 第3步：波段位置判断

三个子条件同时满足：
- 飞吻：0 < DIF < 0.5（在零轴上方但贴零轴）
- 排除高位回调：DIF从近40根最高点的回落幅度 < 50%（区分"刚上零轴的飞吻"和"从高位跌回的假飞吻"）
- 没有高位死叉：最新K线如果出现DIF下穿DEA且DIF仍 > 0.5，则排除

#### 第4步：ST股票过滤

对通过前三步的股票，调通达信 `get_stock_info` 查名称，名称含"ST"的标记为未通过。

### 命令行入口

```
python cassa.py screener                      # 全市场扫描（默认）
python cassa.py screener --pool-size 20       # 只跑K线最多的20只
python cassa.py screener --debug              # 输出每只股票详细筛选过程
python cassa.py screener --min-kline 60       # 最小K线数（默认60）
```

### 输出

- 控制台：摘要表格（扫描数/底背离通过/趋势反转通过/最终通过 + 通过列表 + 未通过列表）
- JSON文件：`result/screener_YYYYMMDD_HHMMSS.json`，每只股票含 code、passed、fail_reason、kline_count、latest_close、latest_date、latest_dif/dea/macd、divergence_found、reversal_confirmed、band_position_ok、divergence_low_date

## 调参过程与结果

### 各轮调参效果

| 版本 | 底背离 | 趋势反转 | 波段位置 | 最终通过 | 说明 |
|------|--------|----------|----------|----------|------|
| v1（初始） | 504 | 3 | 0 | 0 | MACD低点匹配窗口太窄，1640只找不到对应MACD低点 |
| v2（优化背离） | 899 | 7 | 0 | 0 | 直接取价格低点位置MACD值比较，加时效约束30根 |
| v3（放宽节奏） | 899 | 90 | 0 | 0 | 节奏打破从"高于"改为"不低于" |
| v4（跳过波段） | 899 | 90 | - | 90 | 暂时跳过波段位置，观察前两步结果 |
| v5（最近低点优化） | 1788 | 9 | 0 | 0 | 最近低点改为取最近10根最低价，时效约束放宽到50根 |
| v6（简化波段） | 1788 | 9 | 0 | 0 | 波段简化为飞吻(DIF<0.5)+排除高位回调+无高位死叉 |

### 关键调参决策

1. **底背离MACD匹配**：从"要求MACD也形成pivot low且在±5根内"改为"直接取价格低点对应位置的MACD值"，消除了1640只"未找到对应MACD低点"的问题
2. **底背离时效性**：最初没有时效约束，会找到很久以前的背离；加了30根约束后又太严；最终放宽到50根
3. **最近低点检测**：pivot low需要右侧5根K线确认，导致最晚只能找到6月26日的低点；改为取最近10根K线最低价，能覆盖到最新K线
4. **节奏打破**：从"最近20根最低价 > 背离低点"改为"≥ 背离低点"，因为刚见底的股票价格还没涨起来，严格大于几乎不可能满足
5. **波段位置简化**：去掉了盘整判断（振幅<8%太苛刻）和飞吻比例计算（DIF < 近期最大值的30%在刚上零轴时失效），飞吻改为绝对值判断 DIF < 0.5

### 当前结果分析

全市场5488只扫描约18-30秒。最终选出0只，原因是9只趋势反转通过的股票全部卡在波段位置的高位回调——这些股票DIF曾经到过较高位置然后大幅回落到0附近，虽然满足飞吻（DIF<0.5）但回落幅度超过50%，被正确排除。这不是代码bug，是当前市场状态下没有同时满足所有条件的股票。

## 代码结构

选股相关代码在 `cassa.py` 中，按项目分层组织：

- **常量区**：`SCREENER_*` 系列常量（阈值参数）
- **数据结构层**：`KlineBar`、`PivotLow`、`MacdResult`、`ScreenResult`
- **通达信采集层**：`TdxClient.calculate_macd_batch`（批量MACD计算）
- **选股筛选区**（第六层补充）：
  - `load_stock_codes_from_db` - 从SQLite读取股票池
  - `load_daily_kline_from_db` - 读取单只股票K线
  - `align_macd_with_kline` - MACD与K线按日期对齐
  - `find_pivot_lows` - 检测枢轴低点
  - `check_bottom_divergence` - 底背离检测
  - `check_trend_reversal` - 趋势反转确认
  - `check_band_position` - 波段位置判断
  - `screen_single_stock` - 单只股票完整筛选
- **命令执行层**：`run_screener`、`print_screener_summary`
- **参数解析层**：`screener` 子命令

## 未决问题与后续方向

1. **选不出股票**：当前三步筛选条件较严格，全市场0只通过。可考虑放宽高位回调阈值或加入LLM辅助判断
2. **LLM辅助判断**：原计划第6步用LLM判断压力位和突破前信号，第一版暂未实现
3. **波段位置阈值**：飞吻DIF阈值0.5是经验值，不同股价的DIF绝对值差异大，可能需要按股价调整
4. **止盈止损**：视频未涉及，需要从其他来源补充
5. **压力位识别**：视频未给出明确方法，需要额外定义或用LLM判断
6. **分支未合并**：代码在 `feat/screener-v1` 分支，尚未合并到 main
