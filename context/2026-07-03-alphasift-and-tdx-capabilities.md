# AlphaSift 本地化 / 历史选股 / 通达信能力摸底

## 主题概述

这一组上下文来自更早的探索，主要围绕三件事：

1. `alphasift-fork` 如何适配本地数据与历史回放
2. 板块热度、选股、快照、本地日线增强等能力是怎么一步步摸出来的
3. 通达信本地接口在板块、情绪、历史数据方面到底能提供什么

这些内容虽然不是 `Cassa` 的直接实现，但能帮助后续避坑。

## AlphaSift 本地化与热点板块探索

更早的工作里，围绕“最近半个月最热板块”做过一条探索链路，大致经历了：

1. 从在线数据尝试开始
2. 转向本地文本/本地缓存
3. 再落到本地 SQLite 增量同步和统计

这个过程中，逐步形成过：

1. `hotspots-period`
2. `hotspots-period-sync`
3. 本地板块数据库与统计逻辑

这说明一个经验：A 股热点/板块这类能力，如果要稳定复用，最后还是需要本地沉淀和缓存，不适合每次全靠在线即取。

## screen-local 与本地快照链路

后来在 `alphasift-fork` 里还围绕 `screen-local` 做过多轮排查和优化，重点包括：

1. 本地 snapshot 缓存逻辑
2. `auto/refresh` 读取方式
3. 为什么本地选股时常常一个都选不出来

排查得到过一些关键结论：

1. 有时快照源本身返回并不完整
2. 本地 SQLite 日 K 覆盖不完整，会影响增强层
3. 筛选漏斗的每一层都可能把候选快速压缩掉
4. `daily-enrich-max-candidates`、`max-output`、`full-pass` 等参数会明显影响排查体验

还做过一个方向：

- 在选股解释里加入更清楚的中文漏斗说明，以及本地数据自检结果

这类经验对 `Cassa` 很有价值，因为它说明：

1. 数据问题会直接伪装成策略问题
2. 解释链路必须尽量透明

## 历史回放 / 历史选股改造

在“确认选股项目”的会话里，还做过 `screen-local` 历史模式的最小改造，核心是：

1. 增加 `--as-of-date YYYY-MM-DD`
2. 读取指定日期的历史 snapshot
3. 本地 SQLite 日 K 只截取到 `trade_date <= as_of_date`
4. 历史模式下禁用盘中补丁
5. 如果历史 snapshot 缺失，不回退到实时数据

涉及过的关键文件包括：

- `alphasift-fork/alphasift/cli.py`
- `alphasift-fork/alphasift/local_snapshot.py`
- `alphasift-fork/alphasift/daily.py`
- `alphasift-fork/alphasift/tdx_daily_db.py`
- `alphasift-fork/alphasift/pipeline.py`
- `alphasift-fork/alphasift/config.py`

这个方向的价值在于：

1. 以后如果要复盘“某天为什么没选出来”，历史模式是基础能力
2. 历史选股不能偷偷混入实时数据，否则结论会失真

## 通达信能力摸底结论

围绕通达信本地接口，还做过比较系统的能力摸底，形成了几个关键认识。

### 1. 历史 K 线能力

基本确认：

1. 个股历史 K 线可以拿
2. 板块/指数历史 K 线也可以拿

这意味着：

- 用 60 日 / 120 日维度观察主线、轮动、趋势背景，是有数据基础的

### 2. 板块成分能力

基本可用的相关接口思路包括：

1. `get_sector_list`
2. `get_stock_list_in_sector`
3. `get_relation`

因此：

- 板块 -> 成分股
- 股票 -> 相关板块

这两类映射是有条件做的，但历史某一天的板块成分未必能直接按日期回放，需要自己做日度快照沉淀。

### 3. 情绪 / 题材 / 热门文件

还摸到过这些能力入口：

1. `download_file`
2. `get_pricevol`
3. 若干和情绪、热门榜单、讨论热度相关的本地文件

结论不是“有一个完美统一接口”，而是：

- 通达信给了很多零散积木，需要自己拼成“市场温度 / 板块热度 / 热门股 / 题材活跃度”这一层认知。

### 4. 一个重要的现实结论

官方接口里大概率没有一个“带历史日期参数的完整板块热度接口”，所以如果以后要做历史复盘、历史预期、历史盯盘回放，最好自己每天落一份市场快照。

## 对 Cassa 的启发

这些旧探索给新项目的启发主要是：

1. 不要把策略问题和数据问题混在一起
2. 重要市场上下文应该尽量沉淀为本地结构化数据
3. 历史复盘能力很重要，且不能混入实时数据
4. 板块、情绪、主线判断很难靠单一接口一次拿全，必须接受“自己拼装”
