## 2026-07-14 更新：修复 thises 输出中个股 name 为空的问题

### 问题现象

执行：

```powershell
python business.py thises --codes 002317
```

输出结果中：

```json
"name": ""
```

但同一个股票通过 `data.py` 查询基础信息时，通达信接口可以返回名称：

```powershell
python data.py get_stock_info --code 002317.SZ
```

其中包含：

```json
"Name": "众生药业"
```

### 原因

`thises` 当前只输出日 K 数据。

在 `collect_thises_data()` 中，名称直接来自：

```python
target.get("name", "")
```

但是个股在 `resolve_report_code()` 阶段只做代码规整，不会查询股票名称。

因此个股目标会变成：

```json
{
  "raw_code": "002317",
  "target_type": "stock",
  "code": "002317.SZ",
  "name": ""
}
```

板块名称不受影响，因为板块目标会从板块列表中带出 `name`。

### 影响范围

影响范围主要是 `thises` 输出给后续量价分析 Skill 的 JSON。

已知影响：

1. 个股 `name` 为空。
2. `coulling-volume-price-analysis` skill 接收到的数据缺少股票名称上下文。
3. 最终量价分析结果中可能只能展示代码，无法展示股票名称。

不影响：

1. 日 K 数据本身。
2. 板块名称。
3. `report` 输出，因为 `report` 会通过实时数据采集逻辑补名称。

### 修改思路

`thises` 仍然保持只输出日 K 数据。

但在收集日 K 数据时，额外补齐分析对象名称：

1. 如果 `target` 已经有 `name`，直接使用。
2. 如果 `target.name` 为空，则调用 `data.get_stock_info(code, field_list=[])`。
3. 从返回结果中读取 `Name`。
4. 只把名称写入 `thises` 输出，不额外输出完整 `stock_info`。

### 原来代码

```python
    return {
        "raw_code": target.get("raw_code", ""),
        "target_type": target.get("target_type", ""),
        "code": code,
        "name": target.get("name", ""),
        "daily_kline": collect_daily_kline_for_report(code),
    }
```

### 修改后代码

```python

    """收集 thises 量价分析所需的日 K 数据。"""

    name = str(target.get("name", "") or "").strip()

        "raw_code": target.get("raw_code", ""),
        "target_type": target.get("target_type", ""),
        "code": code,
        "name": name,
        "daily_kline": collect_daily_kline_for_report(code),
    }
```

## 2026-07-14 更新：thises 新增筹码分布并调整 data_type

### 原因

2. 当前价格处于成本区上方、下方还是内部。

4. 哪些位置更可能是阻力位。

```python
def add_volume_ratio_to_daily_kline(daily_kline):
    """给日 K 增加 volume_ratio，口径为当日成交量 / 前 5 日平均成交量。"""
    result = []
    for index, row in enumerate(daily_kline or []):
        item = dict(row)
        current_volume = safe_float(item.get("volume"), 0.0)
        previous_rows = daily_kline[max(0, index - 5):index]
        previous_volumes = [
            safe_float(previous_row.get("volume"), 0.0)
            for previous_row in previous_rows
            if safe_float(previous_row.get("volume"), 0.0) > 0
        ]

        if len(previous_volumes) < 5 or current_volume <= 0:
            item["volume_ratio"] = None
        else:
            average_volume = sum(previous_volumes) / len(previous_volumes)
            item["volume_ratio"] = round(current_volume / average_volume, 6) if average_volume > 0 else None

        result.append(item)

    return result
```

## 2026-07-15 更新：调整量价分析输出 data 结构并新增 chip 模块

### 原因

当前 `coulling-volume-price-analysis` skill 的输出结果中，各模块的 `data` 只记录日期和证据文本。

这种结构的问题是：

1. 结论虽然有证据，但无法直接看到完整原始 K 线。
2. 后续复盘或程序消费时，还需要回查输入数据。
3. `chip` 模块需要同时引用筹码分布和相关 K 线证据，仅使用 `trade_date + evidence` 不够。

因此需要把输出证据统一升级为：

```text
单根日 K 完整数据 + evidence
```

同时新增：

```text
chip
```

模块，用来结合 120 日 K 数据和筹码分布数据分析：

1. 主力筹码大多在什么区间。
2. 是否存在套牢盘。
3. 套牢盘筹码大概在什么区间。

### 方案

`direction`、`truth`、`stage`、`smart_money`、`reversal`、`key_price` 中的 `data` 统一改为数组。

数组中每一项都包含：

```json
{
  "kline": {},
  "evidence": ""
}
```

其中 `kline` 是完整单根日 K 数据。

`chip` 模块作为新增顶层模块。

`chip` 模块的数据结构与其他模块不同，因为它需要同时包含：

1. 筹码分布数据。
2. 用于支撑判断的 K 线证据。

其中 `kline_evidence` 也必须使用：

```json
{
  "kline": {},
  "evidence": ""
}
```

## 2026-07-15 更新：新增 market_context 盘中/盘外上下文方案

### 原因

量价分析需要知道当前数据是盘中数据还是盘后完整数据。

如果当前是盘中，`daily_kline` 的最后一根 K 线不是完整日 K，而是盘中临时 K。

这会影响下面这些判断：

1. 成交量是否放大。
2. 收盘价是否站稳。
3. 上下影线是否定型。
4. K 线实体是否定型。
5. 突破是否确认。
6. 反转是否确认。
7. 放量止涨 / 放量止跌是否确认。

如果 LLM 不知道当前是盘中，就可能把盘中累计成交量当作全天成交量，把盘中当前价当作最终收盘价，从而误判量价关系。

### 方案

在 `python business.py thises --codes <codes>` 输出 JSON 顶层增加：

```json
"market_context": {
  "as_of": "日期时间",
  "is_intraday": true
}
```

字段说明：

```text
as_of        当前数据生成时间
is_intraday true 表示盘中，false 表示非盘中
```

`market_context` 放在顶层，不放在每个 item 中。

原因是同一次命令中的所有股票或板块共享同一个盘中 / 盘外状态。

### business.py 完整代码草案

## 2026-07-15 更新：最新一根 K 线 volume_ratio 使用通达信官方量比

### 原因

`thises` 当前计划在每根日 K 中加入：

```text
volume_ratio
```

历史 K 线可以使用：

```text
当日成交量 / 前 5 日平均成交量
```

但最新一根 K 线需要特殊处理。

原因是：

1. 如果当前是盘中，最新一根 K 的成交量只是盘中累计成交量，不是全天成交量。
2. 如果继续用“盘中累计成交量 / 前 5 日平均成交量”计算，容易低估最新 K 的量能。
3. `report` 当前使用通达信 `more_info.fLianB` 作为量比口径。
4. 为了和 `report` 保持一致，最新一根 K 的 `volume_ratio` 应使用 `more_info.fLianB`。

### 方案

`daily_kline[].volume_ratio` 的口径调整为：

```text
非最新 K：当日成交量 / 前 5 日平均成交量
最新 K：通达信 more_info.fLianB
```

不管当前是盘中还是非盘中，只要 `more_info.fLianB > 0`，就用它覆盖最新一根 K 的 `volume_ratio`。

本次不需要修改 `coulling-volume-price-analysis` skill。

原因是 skill 只消费输入中的：

```text
daily_kline[].volume_ratio
```

至于 `volume_ratio` 如何生成，是 `business.py thises` 的数据准备逻辑。

### 完整代码

保留原来的历史 K 量比计算函数：

```python
def add_volume_ratio_to_daily_kline(daily_kline):
    """给日 K 增加 volume_ratio，口径为当日成交量 / 前 5 日平均成交量。"""
    result = []
    for index, row in enumerate(daily_kline or []):
        item = dict(row)
        current_volume = safe_float(item.get("volume"), 0.0)
        previous_rows = daily_kline[max(0, index - 5):index]
        previous_volumes = [
            safe_float(previous_row.get("volume"), 0.0)
            for previous_row in previous_rows
            if safe_float(previous_row.get("volume"), 0.0) > 0
        ]

        if len(previous_volumes) < 5 or current_volume <= 0:
            item["volume_ratio"] = None
        else:
            average_volume = sum(previous_volumes) / len(previous_volumes)
            item["volume_ratio"] = round(current_volume / average_volume, 6) if average_volume > 0 else None

        result.append(item)

    return result
```

新增函数：

```python
def apply_latest_official_volume_ratio(daily_kline, more_info):
    """用通达信官方量比 fLianB 覆盖最新一根 K 线的 volume_ratio。"""
    result = [dict(row) for row in daily_kline or []]
    if not result:
        return result

    official_volume_ratio = safe_float((more_info or {}).get("fLianB"), 0.0)
    if official_volume_ratio > 0:
        result[-1]["volume_ratio"] = round(official_volume_ratio, 6)

    return result
```
