## 第一小步：收集所有数据

### 目标

新增一个用于 `thises` 的数据收集函数。

这个函数的职责是：

1. 接收股票或板块 `code`。
2. 获取后续 `thises` 分析需要的全部数据。
3. 返回 JSON 结构的数据结果。

### 实现原则

当前不重新设计新的数据采集链路。

数据采集逻辑可以复用 `report` 里已经存在的底层收集函数。

接收股票或板块 `code`，并判断输入是板块还是股票的逻辑，也可以直接复用 `report` 现有逻辑。

也就是说：

```text
thises 的数据收集函数
  -> 复用 report 的 code 解析逻辑
  -> 复用 report 底层数据收集函数
  -> 返回 thises 自己的数据 JSON
```

### CLI 入口

新增一个 CLI 入口，用来触发 `thises` 的数据收集流程。

第一版 CLI 入口先只负责：

1. 接收股票或板块 `code`。
2. 调用数据收集函数。
3. 输出收集到的 JSON 数据。

# Thises 量价关系分析 Skill

## 适用场景

使用本 skill 帮用户完成 `thises` 第一版中的量价关系分析。

本 skill 不直接调用底层数据接口，不手写数据采集逻辑，只通过 `business.py thises` CLI 获取数据。

## 必须调用的命令

在 Cassa 项目根目录执行：

```powershell
python business.py thises --codes <codes>
```

其中 `<codes>` 来自用户输入，不能写死。

示例：

```powershell
python business.py thises --codes 881394
python business.py thises --codes 002185
python business.py thises --codes 600519,000001
```

## 输入协议

本 skill 的输入不是用户手写 JSON，而是上面 CLI 命令执行后的 JSON 输出。

Agent 必须先运行命令，再基于命令返回的数据进行量价分析。

## 业务流程

1. 读取用户给出的股票或板块 `code`。
2. 在 Cassa 项目根目录运行 `python business.py thises --codes <codes>`。
3. 读取命令输出的 JSON。
4. 从 JSON 的 `items` 中逐个取出股票或板块的所有日 K 数据。
5. 调用 `skills/coulling-volume-price-analysis` skill，并把日 K 数据传给它。
6. 基于该 skill 的分析结果输出量价分析结果。
7. 如果命令返回 `errors`，需要在结果中说明哪些 code 没有成功分析。

输出结果按照 `coulling-volume-price-analysis` skill 的输出结果来输出。
```

## 第三小步：thises CLI 只输出日 K 数据

### 调整原因

`thises-volume-price` 第一版只做量价分析。

量价分析实际需要传给 `coulling-volume-price-analysis` skill 的核心数据是日 K 数据。

因此，`python business.py thises --codes <codes>` 不需要输出完整数据包。

### 新的输出范围

命令：

```powershell
python business.py thises --codes 881394
```

应只输出日 K 数据。

不再输出：

```text
market_snapshot
stock_info
more_info
relation
macd
chip
```

但建议保留必要的目标信息，方便后续 skill 知道当前分析对象是谁。

建议输出结构：

```json
{
  "task": "thises",
  "data_type": "daily_kline",
  "created_at": "2026-07-14 00:00:00",
  "items": [
    {
      "raw_code": "881394",
      "target_type": "sector",
      "code": "881394.SH",
      "name": "",
      "daily_kline": []
    }
  ],
  "errors": []
}
```

### 对 Skill 业务流程的影响

`thises-volume-price` skill 的业务流程需要相应调整。

原先流程是：

```text
读取命令输出的 JSON。
从 JSON 的 items 中逐个取出股票或板块的所有日 K 数据。
调用 coulling-volume-price-analysis skill，并把日 K 数据传给它。
```

调整后流程是：

```text
读取命令输出的日 K JSON。
从 JSON 的 items 中逐个取出 daily_kline。
调用 coulling-volume-price-analysis skill，并把 daily_kline 以及 code / name / target_type 一起传给它。
```

### 代码调整草案

后续真实代码中，`thises` 数据收集函数可以调整为只收集日 K。

批量构建函数调整为：

```python
def build_thises_data(codes):
    """根据股票或板块 code 批量构建 thises 日 K 数据。"""
    targets = resolve_report_codes(codes)
    items = []
    errors = []

    for target in targets:
        try:
            items.append(collect_thises_data(target))

## 第一小步完整代码草案

以下代码后续可以合入 `business.py`。

当前代码目标只有一个：

```text
跑通 thises 的数据收集入口
```

### 数据收集函数

当前 thises 需要的数据字段与 report 完全一致，直接复用 `collect_report_item`，避免代码重复。

```python
def collect_thises_data(target):
    """收集 thises 所需的全部数据。当前直接复用 report 的采集逻辑。"""
    return collect_report_item(target)
```

### 批量构建数据

```python
def build_thises_data(codes):
    """根据股票或板块 code 批量构建 thises 数据。"""
    targets = resolve_report_codes(codes)
    items = []
    errors = []

    for target in targets:
        try:
            items.append(collect_thises_data(target))
        except Exception as exc:
            errors.append(
                {
                    "raw_code": target.get("raw_code", ""),
                    "code": target.get("code", ""),
                    "target_type": target.get("target_type", ""),
                    "error": str(exc),
                }
            )

    return {
        "task": "thises",
        "created_at": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        "items": items,
        "errors": errors,
    }
```

### CLI 执行函数

```python
def run_thises(args):
    """执行 thises 子命令。"""
    codes = [code.strip() for code in args.codes.split(",") if code.strip()]
    print_json(build_thises_data(codes))
```

### CLI 入口

在 `main()` 中新增（与 `report` 保持一致，使用 `set_defaults(handler=...)` 模式）：

```python
thises_parser = subparsers.add_parser("thises", help="收集 thises 所需数据")
thises_parser.add_argument(
    "--codes",
    required=True,
    help="股票或板块代码，多个用逗号分隔，例如 600519,000001,880675.SH",
)
thises_parser.set_defaults(handler=run_thises)
```
