# 股票池管理：第一版方案

## 主题概述

这份上下文记录的是 `T035` 第一版的目标和边界。

当前确认：先不要把股票池做成复杂系统，也先不做板块池、提醒规则、交易持仓联动，而是先做一个最小可用的“股票入池账本”。

第一版只解决两件事：

1. 股票入池
2. 维护入池理由

## 第一版目标

第一版股票池的定位是：

- 本地股票池管理层
- 观察名单
- 带理由的记录表

它现在不是：

- 完整交易系统
- 提醒系统
- 板块池系统
- 历史事件流水系统

## 当前确认的最小字段

第一版单表字段已经确认如下，字段顺序也已确认：

1. `code`
2. `name`
3. `status`
4. `reason`
5. `create_time`
6. `industry`
7. `concept`

## 字段口径

### `code`

- 使用纯数字股票代码
- 不存 `.SH` / `.SZ` / `.BJ`
- 与项目内部现有股票代码规范保持一致
- 如果后续要调用通达信接口，再在接口边界转换成带后缀代码

### `name`

- 股票名称
- 作为入池时的名称快照保存

### `status`

第一版只允许两个值：

- `观察`
- `丢弃`

其中：

- `观察`：当前仍在池中，继续跟踪
- `丢弃`：放弃观察，但不删除记录

### `reason`

- 自由文本
- 用于记录入池理由
- 第一版不要求结构化标签

### `create_time`

- 记录这条股票池记录第一次创建的时间
- 第一版不额外增加 `update_time`

### `industry`

- 行业字段
- 第一版作为入池时的行业快照保存

### `concept`

- 概念字段
- 第一版先作为文本保存，不拆多表
- 如果后续有多个概念，可以先用文本形式承载

## 当前确认的行为规则

### 1. 单表，不做历史流水

第一版明确使用单表，不额外做“事件流水表”。

这意味着：

- 当前更像“股票池当前状态表”
- 不保留完整的多次入池历史
- 后续如有必要，再扩展历史流水层

### 2. `code` 唯一

第一版建议按 `code` 唯一处理。

### 3. 重复入池规则

如果同一只股票再次入池：

- 若原状态是 `观察`：更新当前记录
- 若原状态是 `丢弃`：恢复为 `观察`

当前讨论中没有要求保留多次入池流水，因此第一版按“更新当前记录”理解。

### 4. 丢弃不删除

当股票不再跟踪时：

- 只把 `status` 改成 `丢弃`
- 不从表中删除

这也是第一版和普通临时列表最重要的区别。

## 为什么先加 `industry` 和 `concept`

讨论中确认，即使第一版暂时只做个股层，也要预留和板块/大盘结合的空间。

因此先把：

- `industry`
- `concept`

直接放进单表里。

这样后面即使暂时不做结构化联动，也至少可以先做：

- 按行业看池子分布
- 按概念看池子分布
- 辅助人工筛选与复盘

## 第一版暂不做的内容

当前明确先不做：

1. 板块池
2. 提醒规则引擎
3. 复杂状态机
4. 交易持仓联动
5. 历史事件流水
6. 结构化 reason tags
7. GUI

## 实现结论

第一版已在分支 `feat/stock-pool-v1` 完成，实现方式与最初方案有调整：

### 入口

- 通达信自定义板块 `CASSA` 作为股票池入口
- CLI 命令：`python cassa.py stock_pool sync`
- 用户在通达信客户端手动把看好的股票加入 CASSA 板块，然后跑 sync 同步

### 存储

- 本地 SQLite：`data/stock_pool.db`
- 单表 `stock_pool`，`id` 自增主键，`code` 可重复（每次导入都是新行）
- 字段：`id / code / name / status / reason / create_time / industry / concept`
- `reason` 第一版统一留空

### 同步规则

- TDX 有、本地无（status=观察）→ INSERT（status=观察，reason 空）
- 本地 status=观察、TDX 已删除 → UPDATE status=丢弃
- 两边都有 → 跳过
- 已丢弃的记录不重新激活（即使 TDX 又加回来也不会恢复，因为只比对 status=观察 的行）

### 与最初方案的差异

1. `code` 不再唯一——允许同 code 重复入池，每次都是新行
2. `reason` 先空着——第一版不做理由录入
3. 入口从"手动 add"改为"通达信自定义板块同步"——用户在通达信里管理股票，程序负责同步
4. 重复入池时不修改历史数据——只看当前 status=观察 的行做比对

### 新增的 TdxClient 方法

- `create_sector(block_code, block_name)`：创建通达信自定义板块（尚未有 CLI 入口）

### 后续待扩展

- 批量入池理由录入
- 板块池
- 提醒规则引擎
