# Screen 选股层第一版方案

## 主题概述

本文件记录 2026-07-09 围绕 Cassa 新版选股层的设计。

用户明确要求新增一个独立选股层：

```text
screen.py
```

新版选股层不沿用旧 `cassa.py screener` 的单体流程，也不围绕旧 screener 的策略注册表继续扩展。它应服务新的数据中心 `data.py`，并允许人或 Agent Skill 用普通 Python 函数灵活组合筛选条件。

## 已确认结论

### 1. 文件命名

选股层入口文件命名为：

```text
screen.py
```

### 2. 一层筛选就是一个函数

选股策略不是一个大函数，也不强制使用 `LAYERS` 注册表。

一层筛选就是一个普通 Python 函数。

标准函数形态：

```python
def filter_xxx(stock_codes):
    """执行某一层筛选。"""
    return remaining_stock_codes
```

入参：

```text
上一层剩下的股票代码列表
```

出参：

```text
这一层筛完后剩下的股票代码列表
```

示例：

```python
stock_codes = get_all_stock_codes()

stock_codes = filter_condition_1(stock_codes)
stock_codes = filter_condition_3(stock_codes)
stock_codes = filter_condition_4(stock_codes)
```

如果要去掉条件 2，就不调用条件 2 对应函数。

如果要新增条件 4，就在本次执行中临时构造 `filter_condition_4` 并调用。

### 3. 策略编排由 Skill 控制

新版选股的核心不是配置系统，而是让 Skill 控制函数调用顺序。

Skill 可以：

1. 选择调用哪些稳定筛选函数。
2. 调整函数调用顺序。
3. 跳过某一层条件。
4. 在本次执行中临时构造一个新的条件函数。
5. 调用 `screen.py` 提供的工具函数打印每层筛选统计。

### 4. 临时条件不写入正式策略 py

用户明确确认：临时新增函数“用完就丢”，不写入正式策略 py，不存在“跑完再删代码”的说法。

因此：

1. 稳定、反复使用、用户确认沉淀的条件，可以进入正式策略脚本。
2. 临时实验条件只服务本次执行，由 Skill 在本次执行过程中临时构造和调用。
3. 临时条件不落入 `strategies/`。
4. 第一版不要围绕“临时写一个 py 再删除”设计流程。

### 5. 股票代码口径

继承数据中心口径：

1. `data.py` 内部使用带后缀通达信代码。
2. `screen.py` 和策略函数也统一使用带后缀股票代码。
3. 业务入口或 Skill 如果接收到纯数字代码，应在业务层补后缀后再进入筛选流程。

示例：

```text
000001.SZ
600519.SH
```

### 6. 初始股票池

选股流程默认从全部 A 股开始。

第一层输入来自：

```python
data.get_stock_list()
```

整理成：

```python
["000001.SZ", "000002.SZ", ...]
```

### 7. 逐层统计

虽然每层函数只返回剩余股票列表，但仍然可以统计每层淘汰数量：

```python
before_count = len(stock_codes)
stock_codes = filter_xxx(stock_codes)
after_count = len(stock_codes)
removed_count = before_count - after_count
```

控制台输出应能看到：

```text
初始股票池：5362
第1层 放量过滤：通过 1200，淘汰 4162
第2层 突破过滤：通过 230，淘汰 970
最终入选：230
```

第一版只要求每层数量统计，不强制记录每只股票的淘汰原因。后续如果需要淘汰原因，再扩展筛选函数返回协议。

3. 临时条件不放进 `strategies/`。

## screen.py 职责

`screen.py` 不负责具体选股条件本身。

`screen.py` 负责：

1. 从数据中心获取初始全 A 股票池。
2. 提供 `run_filter_layer`，执行单层函数并打印统计。
3. 提供 `run_pipeline`，执行由调用方传入的函数流水线。
4. 提供 CLI 入口，运行稳定策略脚本。
5. 输出最终结果到控制台和 `result/`。

`screen.py` 不负责：

1. 不硬编码放量突破、底背离等具体条件。
2. 不使用 `LAYERS` 注册表。
3. 不从自然语言生成策略。
4. 不把临时条件写入正式策略文件。

## 稳定策略脚本写法

稳定策略脚本是普通 Python 文件。

示例：

```text
strategies/volume_breakout.py
```

里面可以定义多个筛选函数：

```python
def filter_volume_expansion(stock_codes):
    return remaining_stock_codes


def filter_breakout(stock_codes):
    return remaining_stock_codes
```

也可以定义一个 `run()` 函数，手工编排调用顺序：

```python
def run(stock_codes, screen):
    stock_codes = screen.run_filter_layer("放量过滤", filter_volume_expansion, stock_codes)
    stock_codes = screen.run_filter_layer("突破过滤", filter_breakout, stock_codes)
    return stock_codes
```

这里 `screen` 是 `screen.py` 模块本身或一个简单上下文对象，目的是让策略脚本复用 `run_filter_layer` 的统计输出。

## 临时条件的执行方式

当用户通过 Skill 要求临时新增条件时，例如：

```text
在放量突破基础上，临时增加条件：最近 5 日涨幅不能超过 20%。
```

Skill 不应把这个条件写入正式 `strategies/volume_breakout.py`。

正确方式：

1. Skill 读取现有稳定策略函数。
2. Skill 在本次执行上下文中临时构造 `filter_recent_gain_limit(stock_codes)`。
3. Skill 按本次用户要求手工编排：

```python
stock_codes = screen.get_all_stock_codes()
stock_codes = screen.run_filter_layer("放量过滤", filter_volume_expansion, stock_codes)
stock_codes = screen.run_filter_layer("突破过滤", filter_breakout, stock_codes)
stock_codes = screen.run_filter_layer("临时过滤：5日涨幅不超过20%", filter_recent_gain_limit, stock_codes)
```

4. 本次执行结束后，不沉淀该临时函数。

## screen.py 完整代码草案

新增文件：

```text
screen.py
```

完整代码草案如下：

```python
"""
Cassa 选股层入口。

第一版采用函数流水线：一层筛选就是一个函数。
每层函数入参为股票代码列表，出参为剩余股票代码列表。
"""

import argparse
import importlib
import json
from datetime import datetime
from pathlib import Path

import data


PROJECT_ROOT = Path(__file__).resolve().parent
RESULT_DIR = PROJECT_ROOT / "result"


def ensure_result_dir():
    """确保结果目录存在。"""
    RESULT_DIR.mkdir(parents=True, exist_ok=True)


def print_json(value):
    """按 JSON 打印结果。"""
    print(json.dumps(value, ensure_ascii=False, indent=2, default=str))


def get_all_stock_codes():
    """从数据中心获取全部 A 股股票代码。"""
    stock_rows = data.get_stock_list()
    stock_codes = []

    for row in stock_rows:
        if isinstance(row, dict):
            code = row.get("Code", "")
        else:
            code = str(row)

        if code:
            stock_codes.append(code)

    return stock_codes


def normalize_filter_output(layer_name, before_codes, after_codes):
    """校验并规整单层筛选函数返回值。"""
    if after_codes is None:
        raise RuntimeError(f"筛选层 {layer_name} 返回 None，应返回股票代码列表")

    if not isinstance(after_codes, list):
        raise RuntimeError(f"筛选层 {layer_name} 返回值不是 list")

    before_set = set(before_codes)
    normalized_codes = []
    for code in after_codes:
        if code in before_set and code not in normalized_codes:
            normalized_codes.append(code)

    return normalized_codes


def run_filter_layer(layer_name, filter_func, stock_codes, layer_records=None):
    """执行一层筛选，并打印通过 / 淘汰数量。"""
    before_codes = list(stock_codes)
    before_count = len(before_codes)

    after_codes = filter_func(before_codes)
    after_codes = normalize_filter_output(layer_name, before_codes, after_codes)
    after_count = len(after_codes)
    removed_count = before_count - after_count

    print(
        f"[选股] {layer_name}: "
        f"输入 {before_count}，通过 {after_count}，淘汰 {removed_count}"
    )

    if layer_records is not None:
        layer_records.append(
            {
                "layer": layer_name,
                "input_count": before_count,
                "passed_count": after_count,
                "removed_count": removed_count,
            }
        )

    return after_codes


def run_pipeline(stock_codes, filter_steps):
    """按传入顺序执行多层筛选函数。"""
    layer_records = []
    current_codes = list(stock_codes)

    print(f"[选股] 初始股票池：{len(current_codes)}")

    for layer_name, filter_func in filter_steps:
        current_codes = run_filter_layer(
            layer_name=layer_name,
            filter_func=filter_func,
            stock_codes=current_codes,
            layer_records=layer_records,
        )

    print(f"[选股] 最终入选：{len(current_codes)}")
    return current_codes, layer_records


def write_screen_result(strategy_name, selected_codes, layer_records):
    """把选股结果写入 result/。"""
    ensure_result_dir()
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    result_path = RESULT_DIR / f"screen_{strategy_name}_{timestamp}.json"

    result_data = {
        "strategy": strategy_name,
        "created_at": timestamp,
        "selected_count": len(selected_codes),
        "selected_codes": selected_codes,
        "layers": layer_records,
    }

    result_path.write_text(
        json.dumps(result_data, ensure_ascii=False, indent=2, default=str),
        encoding="utf-8",
    )
    print(f"[选股] 结果已写入：{result_path}")
    return result_path


def load_strategy_module(strategy_name):
    """加载 strategies 目录下的稳定策略脚本。"""
    module_name = f"strategies.{strategy_name}"
    return importlib.import_module(module_name)


def run_strategy(strategy_name):
    """运行一个稳定策略脚本。"""
    strategy_module = load_strategy_module(strategy_name)
    if not hasattr(strategy_module, "run"):
        raise RuntimeError(f"策略 {strategy_name} 缺少 run(stock_codes, screen) 函数")

    stock_codes = get_all_stock_codes()
    layer_records = []

    # 给策略脚本一个轻量上下文，避免策略直接 import screen 造成循环引用。
    class ScreenContext:
        def run_filter_layer(self, layer_name, filter_func, codes):
            return run_filter_layer(layer_name, filter_func, codes, layer_records)

    print(f"[选股] 策略：{strategy_name}")
    print(f"[选股] 初始股票池：{len(stock_codes)}")

    selected_codes = strategy_module.run(stock_codes, ScreenContext())

    print(f"[选股] 最终入选：{len(selected_codes)}")
    result_path = write_screen_result(strategy_name, selected_codes, layer_records)
    return {
        "strategy": strategy_name,
        "selected_count": len(selected_codes),
        "selected_codes": selected_codes,
        "layers": layer_records,
        "result_path": str(result_path),
    }


def main():
    """命令行入口。"""
    parser = argparse.ArgumentParser(
        description="Cassa 选股层：按 Python 函数流水线执行逐层筛选。",
    )
    parser.add_argument(
        "--strategy",
        required=True,
        help="策略脚本名称，例如 volume_breakout 对应 strategies/volume_breakout.py",
    )
    parser.add_argument(
        "--json",
        action="store_true",
        help="额外打印完整 JSON 结果",
    )
    args = parser.parse_args()

    data.initialize(Path(__file__))
    result = run_strategy(args.strategy)

    if args.json:
        print_json(result)


if __name__ == "__main__":
    main()
```

"""
示例选股策略。

这个文件只演示一层一个函数，以及 run() 中如何手工编排调用顺序。
真实策略由人或 Skill 根据具体选股逻辑编写。

def filter_first_100(stock_codes):
    """示例条件：只保留前 100 只股票。"""

def filter_even_position(stock_codes):
    """示例条件：只保留偶数位置股票。"""

        for index, code in enumerate(stock_codes)
        if index % 2 == 0

# Cassa 选股

使用本 skill 帮用户编排 Cassa 选股流程。

## 核心思路

- 在 Python 代码中沉淀很多小的筛选函数，每个函数只负责一个具体业务条件。
- skill 的主要职责不是重新设计框架，而是控制这些小函数如何调用、何时调用、按什么顺序调用。
- 通过排列组合不同筛选函数，实现不同选股效果。
- 一层筛选就是一个函数。
- 每个筛选函数的入参是股票代码列表，出参也是筛选后剩余的股票代码列表。
- 股票代码在选股层统一使用带后缀通达信格式，例如 `000001.SZ`、`600519.SH`。
- 数据只从 `data.py` 获取，不在策略函数里直接调用 tqcenter 或直接读 SQLite。

## 函数形态

筛选函数保持普通 Python 写法：

```python
def filter_some_condition(stock_codes):
    """根据某个条件筛选股票。"""
    return remaining_stock_codes
```

选股流程就是函数流水线：

```python
stock_codes = get_all_stock_codes()
stock_codes = screen.run_filter_layer("条件1", filter_condition_1, stock_codes)
stock_codes = screen.run_filter_layer("条件3", filter_condition_3, stock_codes)
stock_codes = screen.run_filter_layer("条件4", filter_condition_4, stock_codes)
```

如果用户说去掉条件 2，就不要调用条件 2 对应函数。

如果用户说增加条件 4，就在本次流程中加入条件 4 对应函数。

## Skill 的工作方式

当用户要求选股时：

1. 先理解用户要用哪些筛选条件。
2. 优先复用已有 Python 小函数。
3. 如果缺少某个稳定条件函数，可以新增一个小函数。
4. 按用户要求排列组合这些函数。
5. 用 `screen.run_filter_layer` 执行每一层，保留每层输入数量、通过数量、淘汰数量。
6. 输出最终入选股票列表或结果文件路径。

当用户要求临时新增条件时：

1. 临时条件只服务本次筛选。
2. 不把临时条件写入正式策略文件。
3. 不描述成“改代码再删除”。
4. 本次执行结束后，该临时条件自然消失。

## 输出要求

每次选股都要保留漏斗过程：

- 初始股票数量。
- 每一层条件名称。
- 每一层输入数量。
- 每一层通过数量。
- 每一层淘汰数量。
- 最终入选数量。
- 最终入选股票代码或结果文件路径。

示例：

```text
[选股] 初始股票池：5362
[选股] 放量过滤: 输入 5362，通过 1200，淘汰 4162
[选股] 突破过滤: 输入 1200，通过 230，淘汰 970
[选股] 最终入选：230
```

除非用户要求，不要把选股逻辑讲成复杂框架；优先围绕“小函数如何组合”来执行。
```

## 2026-07-10 已落地实现

第一版 `screen.py` 已开始落地，当前先实现了两个手工验证函数：

```python
def is_box_consolidation(kline_bars, range_max=0.30):
    ...


def is_volume_breakout_from_box(
    box_kline_bars,
    breakout_kline,
    range_max=0.30,
    volume_ratio_min=1.5,
):
    ...
```

当前判断口径：

1. 箱体判断：
   - 传入一段 K 线。
   - 箱体上沿 = 最高价 `max(high_price)`。
   - 箱体下沿 = 最低价 `min(low_price)`。
   - 箱体振幅 = `(上沿 - 下沿) / 下沿`。
   - 当振幅 `<= range_max` 时，判定该区间为箱体。

2. 放量突破判断：
   - 先判断前置 K 线区间是否为箱体。
   - 突破 K 的收盘价必须大于箱体最高价。
   - 突破 K 的成交量必须大于等于箱体平均成交量的 `volume_ratio_min` 倍。

当时 `screen.py` 曾提供两个手工 CLI，以下命令已在后续版本中移除，仅作为历史记录：

```powershell
python screen.py box --code 600289.SH --count 20 --end-date 2026-07-09
python screen.py breakout --code 600289.SH --box-days 20 --end-date 2026-07-09
```

已验证结果：

1. `box` 命令可以正常返回箱体判断及区间指标。
2. `breakout` 命令可以正常返回是否突破、是否放量、箱体上沿、平均成交量、放量倍数等指标。

### screen.py 本次新增/修改代码

#### 1. 新增：获取全部 A 股代码

新增位置：`load_single_stock_kline()` 后

```python

def run_layer(layer_name, input_codes, filter_func, layer_records):
    """执行一层筛选并记录通过数量。"""
    before_count = len(input_codes)
    output_codes = filter_func(input_codes)
    after_count = len(output_codes)
    removed_count = before_count - after_count

    print(

        {
            "layer": layer_name,
            "input_count": before_count,
            "passed_count": after_count,
            "removed_count": removed_count,
        }
    )

    return output_codes

## 2026-07-10 第三次更新：放量突破选股改为直接按突破日取 K 线

### 背景

用户进一步收敛了放量突破的业务逻辑：

```text
对于这个放量突破的选股逻辑，本质就是对于每个股票找一堆K线然后判断：
1. 前面的K线是否是个箱体
2. 最后1根K线是否突破

我们现在先来解决取K线的问题即可。
对于取K线的问题，我们还是按突破日来取，直接用get_market_data这个接口，
end_time填突破日，默认取21条，取K线的数量是在用户传的数量上+1，
然后即可获得一个K线list，然后在做2层筛选。
```

因此不再围绕“突破日前一个交易日”做复杂推导，也不再让 `screen.py` 的放量突破逻辑从本地 SQLite 读取 K 线。

### 最新业务口径

放量突破选股的 K 线获取逻辑统一为：

1. 用户传入 `box_days`，默认 20。
2. 实际拉取 K 线数量为 `box_days + 1`，默认 21。
3. 调用 `data.get_market_data()`。
4. 如果用户传入突破日，则传给 `get_market_data(end_time=突破日)`。
5. 如果用户不传突破日，则 `end_time=""`，由通达信返回最新 K 线。
6. 返回的 K 线列表按交易日升序排列。
7. 前 `box_days` 根作为箱体区间。
8. 最后 1 根作为突破 K。

这个模型下，不需要显式判断“突破日前一个交易日”。

def is_high_pullback_to_ma(
    kline_bars,
    target_index,
    ma_days=DEFAULT_MA_DAYS,
    high_above_ma_ratio=DEFAULT_PULLBACK_HIGH_ABOVE_MA_RATIO,
    ma_tolerance=DEFAULT_MA_TOLERANCE,
):
    """判断目标日是否最高价在 MA 上方、收盘价回到 MA 附近。"""
    moving_average = calculate_moving_average(
        kline_bars=kline_bars,
        target_index=target_index,
        ma_days=ma_days,

        return False

    target_kline = kline_bars[target_index]

    close_price = float(target_kline["close_price"])
    high_distance_ratio = (high_price - moving_average) / moving_average
    close_distance_ratio = abs(close_price - moving_average) / moving_average

    return (
        high_distance_ratio >= float(high_above_ma_ratio)
        and close_distance_ratio <= float(ma_tolerance)
    )

## 2026-07-10 第十次更新：新增增强版箱体判断 v2

### 背景

用户指出当前箱体判断过于粗糙：

```text
现在前面箱体震荡只用最高价最低价的差距来判断我觉得很局限，
很多形态其实都不是一个箱体
```

旧版箱体判断只有一个核心条件：

```text
(区间最高价 - 区间最低价) / 区间最低价 <= range_max
```

这个条件只能说明“波动范围不大”，不能说明价格真的在一个可交易箱体里反复运动。

### 本次原则

先新增 v2，不替换旧函数。

也就是说：

1. `is_box_consolidation()` 保留不动。
2. 当前 `scan-box`、`scan-breakout`、`scan-breakout-pullback-ma5` 暂时仍沿用旧版箱体判断。
3. 新增 `is_box_consolidation_v2()` 和 `analyze_box_consolidation_v2()`，后续人工观察效果后再决定是否切换入口。

### v2 判断思路

增强版箱体不再只看极端最高价和最低价，而是同时看：

1. 稳健上下沿：用高价 90% 分位数和低价 10% 分位数，减少极端影线影响。
2. 箱体振幅：稳健上沿和稳健下沿之间的距离不能太大。
3. 上沿触碰次数：价格需要多次接近上沿，不能只是一根 K 线撑出来。
4. 下沿触碰次数：价格需要多次接近下沿，不能只是一根 K 线撑出来。
5. 收盘价内部比例：大多数收盘价应该落在箱体内部。
6. 中轴漂移：前半段和后半段平均收盘价不能差太多，避免把趋势上涨/下跌误判为箱体。

## 2026-07-10 第十一次更新：箱体判断整体切换为增强算法，并删除 scan-box-v2

## 2026-07-10 第十二次更新：整理控制台输出，JSON 仅 debug 输出

### 背景

用户要求：

```text
整理一下控制台输出， json只需要在--debug模式输出
```

此前 `screen.py` 三个入口都会在末尾直接输出完整 JSON：

```python
print_json(result)
```

这对调试有用，但日常选股时信息太长，不利于阅读。

### 最新输出口径

默认控制台只输出：

1. 数据中心日志。
2. 每层筛选漏斗。
3. 结果摘要。
4. 最多前 100 个入选代码。

只有传入：

```powershell
--debug
```

才输出完整 JSON。

### 新增常量

```python
DEFAULT_CONSOLE_CODE_LIMIT = 100
```

默认摘要最多展示 100 个入选代码。若超过 100 个，会提示：

```text
还有 N 只，使用 --debug 查看完整 JSON
```

### 新增输出函数

```python
def print_screen_result(result, debug=False, code_limit=DEFAULT_CONSOLE_CODE_LIMIT):
    """打印选股结果摘要，debug 模式下追加完整 JSON。"""
    print("[结果] 策略：", result.get("strategy", ""))
    print(f"[结果] 初始股票数：{result.get('initial_count', 0)}")
    print(f"[结果] 入选股票数：{result.get('selected_count', 0)}")

    selected_codes = result.get("selected_codes", [])
    if selected_codes:
        print("[结果] 入选代码：")
        display_codes = selected_codes[:code_limit]
        for index, stock_code in enumerate(display_codes, start=1):
            print(f"[结果]   {index}. {stock_code}")
        if len(selected_codes) > code_limit:
            print(
                f"[结果]   ... 还有 {len(selected_codes) - code_limit} 只，"
                "使用 --debug 查看完整 JSON"
            )
    else:
        print("[结果] 本次没有筛出符合条件的股票")

    if debug:
        print("[DEBUG] 完整 JSON：")
        print_json(result)
```
